项目经历

一、数仓项目

简介：基于大数据平台建设数仓。离线数仓以 Apache Hive+Spark 等实现各层级数仓建设，基于 Druid 和运营工具构建行为分析，用户画像和生意参谋报表等，Impala + Tableau 实现管理和运营应用层报表，主题域主要包括商品、门店、人员、会员等。实时数仓项目以 [ 埋点/日志+Flink+kafka+druid ] 平台工具完成，基于宽表建设指标，并完成看板建设。目前接触过单表亿级以上数据，集群节点20+。

工作内容：

  • 数据接入：数据来源Mysql、日志文件等，基于平台工具完成采集，底层是sqoop+flume/SDK。入库到ODS层保留源数据格式，采集前会进行数据探查，基于python/shell+hive 脚本实现查看数据状况。通过调度中添加采集入库条数相等、不为空规则保证准确性。
  • 模型设计、数据清洗和性能调优：基于 Hive on Spark 或 Impala 完成数据交互式分析、数据清洗和调度，模型主要基于维度建模理论，DWD层主要分主题域，DWS层轻度聚合，DIM维表通过快照处理维度变化。调度通过平台工具完成，底层基于azkaban。关于数据倾斜情况，其中一种情况是由于业务的28法则，大卖家产生大量记录，通过拆分出倾斜键，并扩大表N倍取模再join解决。
  • 数据质量：ODS和DWD层主要制定完整性和一致性规则，常规的主键唯一性，维度枚举，NULL检测等，DWS和ADS主要根据指标制定特定规则，常规的表行数绝对值以及波动率，关键指标值非空和枚举等。通过 python/shell+hive+api 脚本实现监控和告警。
  • 数据清洗和数据字典文档维护。

二、大数据运维管理平台

简介：解决大数据平台的部署、配置和监控复杂性。借鉴 CDH 和HDP 大数据管理平台系统，采用开源 Apache Ambari 完成大数据运维管理平台，实现自动化部署、服务集成、管理（服务启停、配置和多版本管理）和监控等功能，大大提高项目POC和集群管理效率。目前服务多个客户，100+节点。

工作内容：负责自动化部署功能开发，大数据组件服务集成和维护，以及线上问题支持。

  • 自动化部署功能开发：基于 Ambari+Python+Shell 实现 shell 脚本一键部署，功能包含但不限于集群免密，单用户部署，主机参数优化，ntpd服务配置等；依赖组件和数据库的配置、部署和初始化等。以及基于不同客户的自定义服务部署。
  • 服务集成和维护：基于 Ambari 框架二次开发，在stack下模块下完成服务集成。实现 Impala、Astro、Azkaban等服务集成。降低服务部署、使用和维护的门槛，这种服务模块化的集成，高效解决多个客户服务的配置、版本等管理。
  • 技术预研：基于 Jenkins + Git + Shell + vmware 完成搭建一键部署的 CI/CD 流程。调研并协助完成 K8S+Docker（Harbor + Rancher）容器化迁移等。
